## 🧠 Core Architectural Philosophies

My approach is deeply rooted in structured, business-aligned architecture. Some key concepts I advocate for include:

- **The "Lego" Framework**: Building standardized skeletons with company-proven patterns (.NET, REST, SQL) and using Spec-Driven Development (SDD) for modular joints.
- **Hexagonal & Hybrid Microservices**: Leveraging Port and Adapter architecture, DDD, and CQRS for complex domain APIs, balancing idealism with pragmatic enterprise constraints.
- **Modern Standards**: Emphasizing SOLID and CUPID principles, CI/CD readiness, stringent test coverage, and internal package reuse via NuGet and npm feeds.
- **AI as the "Great Teacher"**: Using AI as an engine for the structural 90% tasks, allowing humans to focus on the final 10% of refinement and business alignment.

## 🚀 Recommended Hosting Options for Astro

Since this is an Astro-based static site, it can be hosted for **free**. Given your focus on keeping things simple and integrated with GitHub, here are the two absolute best options:

1. **GitHub Pages (Top Recommendation)**
   - **Cost**: 100% Free.
   - **Why**: Since you rely heavily on your GitHub profile, you can use GitHub Actions to build and deploy your Astro site directly to GitHub Pages automatically every time you push code. It keeps everything strictly within the GitHub ecosystem and is great for growing a portfolio check-in history.
   
2. **Cloudflare Pages**
   - **Cost**: Generous Free Tier.
   - **Why**: If you ever need a highly performant global CDN or advanced edge computing features, Cloudflare Pages seamlessly connects to your GitHub repository and deploys automatically on every push, just like GitHub Pages, but with slightly faster global enterprise-grade delivery.
